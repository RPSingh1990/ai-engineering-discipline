# Code Of Conduct

## Standard

This project is for practical, evidence-based collaboration around AI-assisted engineering governance.

Participants are expected to:

- be respectful and direct
- critique ideas with concrete evidence
- keep examples public-safe
- avoid personal attacks, harassment, and discriminatory language
- avoid posting secrets, private prompts, customer data, or internal repo paths

## Unacceptable Behavior

Unacceptable behavior includes:

- harassment, threats, or abusive language
- publishing private or sensitive data
- submitting intentionally misleading examples
- spamming promotional links unrelated to the project
- using issues or discussions to target individuals or organizations

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or discussions that violate this standard. Repeat or severe violations may result in blocking.

## Reporting

Report conduct issues privately to the repository owner. Include links, screenshots, and a short description of the concern when possible.
